## Architecture

### State mechanism

A sentinel file `~/.cx/disabled` encodes the global disabled state. Its presence means disabled; its absence means enabled. No JSON, no config field, no DB query — just `os.Stat`. The file content is a JSON object:

```json
{"disabled_at": "2026-03-25T10:00:00Z", "reason": ""}
```

The `reason` field is populated by a future `--reason` flag; for v1 it is always an empty string. The timestamp is recorded for audit purposes only.

### File layout

New files:
```
cmd/disable.go
cmd/enable.go
internal/project/disable.go
```

Modified files:
```
cmd/root.go   — PersistentPreRun + AddCommand calls
```

### Data flow: cx disable

```
cx disable
  └── project.IsDisabled()                   check idempotency
  └── project.LoadRegistry()                 get all registered project paths
  └── for each project path:
        └── agents.DetectInstalled(path)      find which agents are installed
        └── for each installed agent:
              └── os.ReadFile(configFilePath)  read current content
              └── project.BackupAgentConfig()  write to ~/.cx/agent-backups/<id>-<ConfigFile>
              └── os.WriteFile(configFilePath) write minimal stub
  └── project.SetDisabled("")                create ~/.cx/disabled
  └── print confirmation
```

### Data flow: cx enable

```
cx enable
  └── project.IsDisabled()                   guard: already enabled → print + return 0
  └── project.ClearDisabled()                remove ~/.cx/disabled
  └── project.RestoreAgentConfigs()          scan ~/.cx/agent-backups/, restore each file
  └── print confirmation + list of restored files
```

### Data flow: PersistentPreRun gating

```
any cx subcommand invocation
  └── rootCmd.PersistentPreRun fires
        └── project.IsDisabled()
        └── if disabled AND cmd.Use not in allowlist {"enable","disable","version"}
              └── fmt.Fprintln(os.Stderr, "cx is disabled. Run 'cx enable' to restore.")
              └── os.Exit(1)
```

The allowlist check uses `cmd.Use` (the leaf command's `Use` field, which is the first token — e.g., `"enable"` from `Use: "enable"`). Cobra passes the leaf command to `PersistentPreRun`, so no tree walking is needed.

## Technical Decisions

### Sentinel file vs. config field

Chosen: sentinel file `~/.cx/disabled`. Rationale: zero-cost to check (single `os.Stat`), no JSON parsing on every command invocation, trivially inspectable by the developer, no risk of corrupting `preferences.json`. Config field would require reading and parsing a JSON file on every command start.

### Backup naming scheme

`~/.cx/agent-backups/<project-id>-<ConfigFile>` — e.g., `~/.cx/agent-backups/abc123def456-CLAUDE.md`.

- `<project-id>` is the 12-character SHA-256 hex prefix of the absolute project path, matching the existing `projectID()` function in `registry.go`.
- `<ConfigFile>` is the bare filename (`CLAUDE.md`, `GEMINI.md`, `AGENTS.md`), not a path.

`ExportProjectID(path string) string` in `internal/project/disable.go` re-implements or calls the existing unexported `projectID()`. The cleanest approach is to **export it** by renaming `projectID` to `ProjectID` in `registry.go` and updating the two call sites in that file.

### Minimal stub content

```
<!-- cx disabled — run `cx enable` to restore. Do not edit this file while cx is disabled. -->
```

A single HTML comment. Invisible to Claude Code as an instruction. Immediately communicates the disabled state to any developer who opens the file. The "do not edit" advisory protects against the edge case where edits are overwritten on `cx enable`.

### Restore behavior for missing projects

When `RestoreAgentConfigs()` encounters a backup whose project path is no longer in the registry (project was deleted or deregistered while disabled), it:
1. Logs a warning to stderr: `warning: project path <path> not found; backup retained at <backup-path>`
2. Does NOT delete the backup (it is retained for manual recovery)
3. Continues restoring remaining backups

### cx disable idempotency

If `~/.cx/disabled` already exists when `cx disable` is run:
- Print: `cx is already disabled.`
- Exit 0.
- No backup overwrite (existing backups are preserved).

### cx enable when not disabled

If `~/.cx/disabled` does not exist when `cx enable` is run:
- Print: `cx is already enabled.`
- Exit 0.

## Implementation Notes

### internal/project/disable.go — full function signatures

```go
// IsDisabled returns true if ~/.cx/disabled exists.
func IsDisabled() bool

// SetDisabled creates ~/.cx/disabled with a JSON timestamp payload.
// reason may be empty.
func SetDisabled(reason string) error

// ClearDisabled removes ~/.cx/disabled. No-op if it does not exist.
func ClearDisabled() error

// BackupAgentConfig writes content to ~/.cx/agent-backups/<projectID>-<configFile>.
// projectPath is the absolute path to the project root.
// configFile is the bare filename (e.g., "CLAUDE.md").
func BackupAgentConfig(projectPath, configFile, content string) error

// RestoredFile describes a single successfully restored agent config.
type RestoredFile struct {
    ProjectPath string
    ConfigFile  string
}

// RestoreAgentConfigs reads all files in ~/.cx/agent-backups/, resolves each
// backup to its target project path via LoadRegistry(), restores the content,
// and deletes the backup on success.
// Returns the list of successfully restored files and any non-fatal warnings.
func RestoreAgentConfigs() ([]RestoredFile, []string, error)
```

Backup directory creation is handled inside `BackupAgentConfig` using `os.MkdirAll` on `~/.cx/agent-backups/`. Use `GlobalCXDir()` to get `~/.cx/` and append `agent-backups`.

### Exporting projectID

In `internal/project/registry.go`, rename `projectID` → `ProjectID` and update both call sites in the same file. `disable.go` then calls `ProjectID(projectPath)` without duplication.

### cmd/root.go changes

Add before the closing brace of `init()`:
```go
rootCmd.AddCommand(disableCmd)
rootCmd.AddCommand(enableCmd)
```

Add as a field on `rootCmd` declaration:
```go
PersistentPreRun: func(cmd *cobra.Command, args []string) {
    if !project.IsDisabled() {
        return
    }
    allowed := map[string]bool{"enable": true, "disable": true, "version": true}
    if allowed[cmd.Use] {
        return
    }
    fmt.Fprintln(os.Stderr, "cx is disabled. Run 'cx enable' to restore.")
    os.Exit(1)
},
```

Import `"github.com/AngelMaldonado/cx/internal/project"` in `cmd/root.go`.

### cmd/disable.go structure

```go
var disableCmd = &cobra.Command{
    Use:   "disable",
    Short: "Suspend cx across all projects",
    RunE:  runDisable,
}
```

`runDisable` should:
1. Check `project.IsDisabled()` → idempotency message + return nil
2. Call `project.LoadRegistry()` to get all project paths
3. For each path, call `agents.DetectInstalled(path)` to get installed agents
4. For each installed agent whose config file exists: read, backup, write stub
5. Call `project.SetDisabled("")`
6. Print summary: number of projects and config files stubbed

### cmd/enable.go structure

```go
var enableCmd = &cobra.Command{
    Use:   "enable",
    Short: "Restore cx across all projects",
    RunE:  runEnable,
}
```

`runEnable` should:
1. Check `!project.IsDisabled()` → idempotency message + return nil
2. Call `project.ClearDisabled()`
3. Call `project.RestoreAgentConfigs()` → print each restored file and any warnings
4. Print summary

### Cobra PersistentPreRun interaction with subcommands

If any subcommand defines its own `PersistentPreRun`, it will replace the root's `PersistentPreRun` by default (Cobra does not chain them automatically). Currently no subcommands define `PersistentPreRun`, so this is not a problem for v1. If this becomes an issue later, use `PersistentPreRunE` with explicit chaining.

### Testing approach

Unit tests in `internal/project/disable_test.go`:
- `TestIsDisabled`: write/remove sentinel, verify bool return
- `TestSetClearDisabled`: round-trip, verify JSON content
- `TestBackupAgentConfig`: write backup file, verify path and content
- `TestRestoreAgentConfigs`: set up mock project registry + backup files, call restore, verify files written and backups deleted

Integration tests in `cmd/disable_test.go` (using `os.Exec` or Cobra's test helpers):
- `TestDisableEnable`: set up temp project with mock registry, run disable, verify stub + sentinel, run enable, verify restoration
- `TestGatingWhileDisabled`: run arbitrary command while disabled, verify exit code 1 and error message
- `TestIdempotency`: double-disable, double-enable
- `TestRestoreMissingProject`: backup exists but project removed from registry, verify warning + backup retained
